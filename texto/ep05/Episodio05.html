Vamos lá!

Através do código binário dois estados para um mesmo objeto, no caso aqui, eu coloquei a lâmpada em uma representação para a gente entender o código binário.

Se você tem uma sequência de lâmpadas, elas acesas e apagadas, nessa sequência, nós podemos representar letras, números e sinais gráficos, e isso vai depender da notação posicional.

O que é a notação posicional? É a posição de cada objeto, de cada lâmpada dessa na sequência, ela vai ter uma numeração nessa posição.

Por exemplo, vai começar da lâmpada zero ou vai até a lâmpada oito, é uma sequência de acender e apagar vai fazer essa representação.

Essa representação tem uma tabela, através da tabela ASCII, que é America Standard

Code for Information Interchange.

Então nós vamos ter dois tipos de codificação, nós vamos ter a codificação binária, vamos ter a codificação decimal e a representação dessa codificação.

Essa representação pode ser uma letra, número, um sinal gráfico ou mesmo uma uma função específica para que o computador possa funcionar.

Vamos conhecer um pouquinho de como é o código binário.

Até andei demonstrando por aqui quando eu fiz uma placa de Arduíno (vídeo no Youtube) com alguns LEDs com oito LEDs mostrando o acender e apagar, e através de cálculos, as lâmpadas subsequentes vão acendendo ou apagando e formando essa essa variação de sequência na tabela ASCII  e consequentemente as representações que ela dispõe.

Quero acrescentar também que cada lâmpada desse nosso nosso conjunto de lâmpadas, é chamado, individualmente, de bit.

Vem do termo inglês binary digit é o conjunto dessas oito lâmpadas é chamado de Byte, também no inglês binary term.

A notação posicional, no nosso caso, consiste em dar um número inteiro a cada lâmpada de acordo com a posição ocupada na sequência.

Fazemos a leitura da direita para esquerda, tendo a primeira lâmpada a posição de valor zero.

A segunda lâmpada terá a posição um, seguindo assim até a última lâmpada, que é a posição número sete.

Vou colocar abaixo de nosso conjunto de lâmpadas o número dois que representa os dois estados possíveis, aceso, apagado, assim nós teremos a representação

dos binários.

Desta forma, poderemos ver a representação de um código, letra,

número ou caractere gráfico na tabela ASCII.

Além do código binário, temos os valores decimais. Para essas mesmas representações

basta elevarmos o número dois que é a base do nosso código, a posição que o bit se encontra.

Lembramos que todo número elevado a zero o resultado é sempre um. Então para o binário acima apresentado temos o seguinte valor decimal correspondente.

Depois de obtermos o resultado da exponenciação, somamos os decimais que tem como correspondente o binário número um.
